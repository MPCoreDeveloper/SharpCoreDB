# SharpCoreDB Compatibility Smoke Tests

**Location:** `tests/CompatibilitySmoke/`  
**Version:** v1.6.0  
**Purpose:** Validate protocol handshake, simple query, and metadata discovery against a live SharpCoreDB Server instance.

---

## Overview

The smoke tests cover three compatibility layers:

| Layer | Tests |
|---|---|
| **HTTP REST API** | Health check, JWT authentication, SELECT 1 query, `information_schema` discovery |
| **Binary Protocol (TCP)** | TCP connect, SSL negotiation (SSLRequest/S handshake), PostgreSQL startup message exchange |
| **Output** | JSON results file (`smoke-results.json`), coloured terminal output, CI-friendly exit codes |

---

## Prerequisites

| Requirement | Notes |
|---|---|
| **.NET 10 SDK** | Required to build and run the server |
| **Python 3.10+** | Required to run `smoke_tests.py` |
| **`requests` package** | `pip install requests` (auto-installed by the PowerShell runner) |
| **`dotnet dev-certs`** | Ships with .NET SDK; used to generate the test TLS certificate |

## Running Locally

### Option A — PowerShell runner (recommended)

From the repository root:

```powershell
pwsh tests/CompatibilitySmoke/run-smoke.ps1
```

The runner will automatically:
1. Build the server in Release mode.
2. Generate a dev TLS certificate at `tests/CompatibilitySmoke/smoke-certs/smoke.pfx`.
3. Write a patched server config with absolute paths.
4. Start the server in the background.
5. Poll `https://127.0.0.1:8443/api/v1/health` until ready.
6. Execute all smoke tests.
7. Stop the server and clean up temp files.

**Options:**

```powershell
# Skip build (server already built)
pwsh run-smoke.ps1 -SkipBuild

# Custom ports
pwsh run-smoke.ps1 -HttpsPort 9443 -PgPort 5555

# Keep the server running after tests (for manual inspection)
pwsh run-smoke.ps1 -KeepServer

# Increase wait timeout for slow CI agents
pwsh run-smoke.ps1 -Timeout 120
```

### Option B — Manual (advanced)

**1. Generate a test certificate:**

```powershell
dotnet dev-certs https -ep tests/CompatibilitySmoke/smoke-certs/smoke.pfx -p smoketest --trust
```

**2. Start the server with the smoke config:**

```powershell
dotnet run --project src/SharpCoreDB.Server -c Release -- `
    --appsettings tests/CompatibilitySmoke/appsettings.smoke.json
```

The server must be fully started before running the tests. Monitor the console until you see:

```
[HH:mm:ss INF] SharpCoreDB Server is ready.
```

**3. Run the Python tests:**

```bash
pip install requests
python tests/CompatibilitySmoke/smoke_tests.py \
    --host 127.0.0.1 \
    --https-port 8443 \
    --pg-port 5433 \
    --username smokeadmin \
    --password admin123 \
    --no-verify-tls \
    --output smoke-results.json
```

## Test Configuration

The default smoke config (`appsettings.smoke.json`) uses:

| Setting | Value |
|---|---|
| Admin username | `smokeadmin` |
| Admin password | `admin123` |
| Test database | `smokedb` (no encryption, temp path) |
| HTTPS port | `8443` |
| Binary protocol port | `5433` |
| TLS cert | `./smoke-certs/smoke.pfx` (generated by runner) |
| System databases | disabled |
| Encryption | disabled (for CI speed) |

> **Security note:** The smoke config is intended for local development and CI only. The credentials and weak TLS settings must never be used in production.

## Troubleshooting

### `Server did not become ready within Xs`

- Ensure no other process is using ports 8443 or 5433.
- Check the server console output for startup errors (cert not found, port in use, missing DB path).
- Try increasing `-Timeout`:
  ```powershell
  pwsh run-smoke.ps1 -Timeout 120
  ```

### `Certificate generation failed`

On Linux, `dotnet dev-certs https --trust` requires root or the `ca-certificates` package. Use `-p smoketest` without `--trust` in headless CI environments — the smoke tests pass `--no-verify-tls` to bypass certificate validation.

### `Binary protocol SSL negotiation` fails with `unexpected response: b'N'`

The server returned `N` (TLS not supported). Confirm `EnableBinaryProtocol: true` and `TlsEnabled: true` in the smoke config.

### `JWT authentication` returns `401`

The password hash in `appsettings.smoke.json` must match the SHA-256 hash of `admin123`. Verify the `smokeadmin` user entry is present in the `Security.Users` array.
